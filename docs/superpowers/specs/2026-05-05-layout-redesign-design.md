# 设计规范：界面风格与布局重构

## 1. 概述
本设计规范基于视觉评估反馈，对应用的侧边栏、顶部导航栏以及整体地图风格进行重新设计。确定的方向为“选项 A：清新明亮风格”。

## 2. 整体视觉风格
- **主色调**：浅蓝和白色搭配（基于 Tailwind 提供的预设色系）。
- **地图区域**：替换深色占位符，采用视觉舒适的浅蓝水域色作为主背景，洲际文字标签采用高对比度深色。

## 3. 左侧菜单栏 (Sidebar) 组件
左侧提取为独立的侧边栏面板，承载应用的主要导航。

### 结构与路由绑定
1. **🏠 首页**
   - 交互：点击无缝跳转至根路由 `/`，返回全景地图页。
2. **🌍 大洲**
   - 角色：不可点击的分类层级标题。
   - 子项：
     - 北美洲 -> 触发路由 `/continent/north-america`
     - 欧洲 -> 触发路由 `/continent/europe`
     - 亚洲 -> 触发路由 `/continent/asia`
3. **🤖 AI 助手**
   - 交互：独立的菜单项，预留后续设计的 AI 交互事件。

## 4. 顶部工具栏 (Header) 组件
废弃原有的冗余文本，统一修改为核心工具入口。

### 功能清单
1. **快速评测**
2. **我的收藏**
3. **系统设置**
*实现说明*：初期实现搭建 UI 框架和鼠标悬停（hover）交互状态，并预留点击事件处理函数。

## 5. 组件级影响
这些布局更改将直接重构 `MapPage.jsx` 中的 `aside` 和 `header` 部分的硬编码代码。
