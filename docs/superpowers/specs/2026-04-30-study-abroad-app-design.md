# 留学推荐软件 - 架构设计方案 (Design Spec)

## 项目概述
本项目是一个用于查找、AI 推荐以及申请海外留学项目的桌面端软件。核心目标是提供一个统一的平台，用户可以在其中浏览爬取到的大学项目数据，根据自身背景获得大语言模型（LLM）的个性化推荐，并且能直接在软件内部的浏览器视图中进行官网申请。

主要约束条件为：**运营成本趋近于 0（白嫖/低成本）**，**支持小圈子内好友共享数据**，以及**强制要求能在软件内部直接打开学校的申请页面进行网申**。

## 架构与技术栈

### 1. 软件外壳与前端展示 (桌面端 App)
- **核心框架**: Electron（提供桌面端运行环境和内置网页渲染能力）。
- **前端技术栈**: React 19 + Vite（复用项目中已有的栈）。
- **核心机制**: 
  - 采用 Electron 的 `BrowserView` 或 `<webview>` 组件，在软件内部直接渲染外部大学的申请网页。这能够完美绕过普通浏览器中常见的跨域限制（CORS）以及 `X-Frame-Options: DENY` 安全策略，实现“软件内直申”的核心需求。

### 2. 数据存储 (云端共享 + 本地辅助)
- **云端数据库**: Supabase (基于 PostgreSQL) - 使用其免费额度。
  - 作为中央数据仓库，存储爬取到的大学项目列表、用户基本画像和收藏记录。
  - 使得多位好友在使用该软件时，能够共享同一个庞大的留学项目数据池。
- **本地存储**: 配合项目现有的 SQLite (`better-sqlite3`)，可以用来在本地缓存数据、存储用户的私密设置（比如用户自己的大模型 API Key），以保证隐私和安全。

### 3. 数据采集 (网络爬虫)
- **爬虫引擎**: Node.js 环境下的 `cheerio` 与 `rss-parser`（已经在 `package.json` 中配置），直接集成在 Electron 的主进程中运行。
- **运行方式**: 爬虫脚本由用户的电脑在本地直接执行。用户可以手动点击“同步/抓取”按钮，软件在后台访问学校官网，清洗提取关键信息（学校名称、项目名称、截止日期、申请要求等），然后统一推送到云端的 Supabase 数据库。这彻底免去了部署云端爬虫服务器的成本。

### 4. AI 推荐引擎
- **接入方式**: 在 Electron 主进程中，直接通过 HTTP 请求调用各大语言模型（如 OpenAI、DeepSeek、Kimi、通义千问等）的 API。
- **工作流**: 
  1. 用户在前端输入自身背景条件（GPA、专业、语言成绩、预算等）。
  2. 软件从 Supabase 数据库中检索出一批符合硬性门槛的项目。
  3. 软件将用户背景和筛选出的项目组合成 Prompt（提示词），发送给 AI 模型。
  4. AI 分析后，返回格式化的推荐列表以及个性化的选校指导建议。

## 模块划分

1. **项目大厅与搜索页**: 用于浏览、过滤和检索 Supabase 中的留学项目。
2. **AI 顾问对话页**: 提供聊天或表单界面，供用户获取个性化的选校推荐。
3. **应用内置浏览器 (内嵌网申系统)**: 这是一个特殊的视图层。当用户点击“立即申请”时，利用 `BrowserView` 弹出一个覆盖层，加载真实的学校官网。它会附带简单的导航栏（如后退、前进、关闭、刷新）。
4. **数据采集管理中心**: 隐藏的高级面板，用于管理爬虫规则、运行抓取任务以及查看日志。

## 权衡与选择原因 (Trade-offs)
- *为什么不选择纯 Web 网页？* 纯网页在试图用 `iframe` 嵌入其他大学网申系统时，会遭遇极大的安全策略拦截，导致页面空白。Electron 可以从底层解决这个问题。
- *为什么不租一台云服务器做后端？* 独立的 VPS 或云服务器每个月都会产生固定开销。使用免费的 Supabase + 客户端本地计算（运行爬虫和调用 AI API）的混合架构，能够在实现数据共享的同时，将服务器成本压至真正的 0 元。

## V1.0 版本验收标准 (Success Criteria)
- [ ] 桌面端应用能够成功打包并运行。
- [ ] 用户可以触发一次本地爬虫抓取，并将数据成功保存到云端数据库。
- [ ] 用户可以浏览项目列表，并成功发起一次 AI 推荐请求，获得结果。
- [ ] 用户点击项目链接后，目标学校的申请系统能够在软件内部以浏览器视图安全、正常地打开。
