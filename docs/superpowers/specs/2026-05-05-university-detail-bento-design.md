# 设计规范：大学详情页 Bento Box 架构

## 1. 概述
本设计规范定义了“阶段二：大学详情页”的 UI 架构。根据用户评估反馈，详情页摒弃了传统的列表/左右分栏式阅读体验，全量采用“Bento Box（便当盒网格）”的现代化卡片式设计语言，以提供信息密度高、视觉冲击力强、且极具高级感的数据展示方案。

## 2. 核心页面结构 (`UniversityPage`)

整个页面采用浅色背景（如 `bg-slate-50` 或 `bg-surface-bright`），主内容区域通过限定最大宽度（如 `max-w-6xl`）居中对齐，内部划分为多个不同尺寸、相互拼接的圆角卡片。

### 2.1 头部名片 (Header Bento)
- **视觉特征**：宽矩形卡片，白色背景。
- **内容元素**：
  - 左侧：大学 Logo (深色/亮色高对比度)、中英文全称、地理位置（州/国家）。
  - 右侧：高亮的 QS 世界排名徽章（如醒目的黄色/蓝色药丸形状）。
  - 交互：返回上一页的后退按钮。

### 2.2 核心多媒体卡片 (Hero Media Bento)
- **视觉特征**：最大尺寸的矩形卡片（跨两列或跨多行），深色半透明遮罩背景。
- **内容元素**：
  - 校园标志性建筑高清图片或轮播视频。
  - 左下角带有悬浮的“校园风采”文字标识及播放 Icon。

### 2.3 关键数据看板 (Stat Bentos)
围绕多媒体卡片，排布若干正方形或小矩形卡片，用于展示关键数字：
- **学费概览卡片**：
  - 顶部：小号灰色字体“平均本科学费”。
  - 中部：巨大的黑色加粗字体展示具体金额（如 `$55,000/年`）。
  - 底部：小型数据可视化元素（如使用 Tailwind 绘制的极简阶梯柱状图，暗示投资回报）。
- **录取率/难度卡片**：
  - 采用中心圆环图（Circular Progress）包裹具体百分比（如 7%），直观展示录取难度。

### 2.4 留学项目入口 (Programs Bento)
- **视觉特征**：底部跨行/跨列的长条形深色卡片（如 `bg-slate-800`）。
- **内容与交互**：
  - 文字：“浏览留学项目 (本/硕/博 120+ 专业)”。
  - 交互：点击后可以触发底部的抽屉滑出（Bottom Sheet）或模态框，展示具体的专业信息，以保证主 Bento 画面的纯净度。

## 3. 数据层级与隔离
- 当前阶段只处理硬编码的 Mock 数据以跑通 UI 表现。
- 组件必须预留清晰的 Props 接口（如 `university.name`, `university.tuition`, `university.ranking`），确保在“阶段三”接入 Supabase 抓取数据时能够无缝替换。

## 4. 技术依赖
- 无需新增外部图表库，所有简单的可视化（如录取率圆环、条形图）均使用 Tailwind 的原生类名及基础 SVG 实现。
- `framer-motion` (可选)：若需增加各个 Bento 块进入时的阶梯弹出动画，可考虑轻量级引入；若不引入则使用 Tailwind 的 `transition` 实现 Hover 微动效。
