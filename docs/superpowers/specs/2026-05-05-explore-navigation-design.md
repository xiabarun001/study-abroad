# 设计规范：探索导航与双模式选校框架

## 1. 概述
本阶段旨在优化应用的地理查找和院校筛选逻辑，引入“双轨制找校设计”。重点解决层级过深的问题，通过右侧滑出式抽屉（Drawer）和网格式院校大厅（Explore Hub），让用户能快速、扁平地查找目标大学。

## 2. 核心交互模式

### 2.1 探索模式（基于地图与侧边抽屉）
- **触发机制**：点击主地图上的大洲标签（如“北美洲”、“欧洲”），或点击左侧菜单栏下方的“大洲”子菜单项。
- **展现形式**：不跳转新路由，从屏幕右侧平滑滑出一个宽大、带有毛玻璃质感（`backdrop-blur`）的 Drawer 面板。
- **内部结构**：
  - **顶部**：显示当前大洲的名称及关闭按钮。
  - **内容区**：瀑布流或列表形式直接展示该洲属国家（作为分组标题）及其热门院校卡片。
  - **交互**：用户可以直接在抽屉内滑动浏览，点击某个卡片即可触发详情展示逻辑。

### 2.2 直达模式（基于筛选大厅）
- **入口**：左侧菜单栏在“首页”下方新增“院校大厅 (Explore)”入口。
- **页面结构**：
  - **视图替换**：点击后，原本的主地图区域将切换为一个全屏的院校卡片网格列表。
  - **筛选控制台**：列表上方提供多维度级联筛选器（洲际筛选、国家筛选、快速字母定位等）。
  - **卡片设计**：每个院校卡片包含其 Logo、中英名称、所在国家及核心优势标签。

## 3. 详情页架构预埋
- **路由交互**：点击任意院校卡片，系统会通过 Modal 弹窗或独立路由 (`/university/:id`) 进入详情界面。
- **模块划分**：
  1. 本硕博项目介绍（Programs）
  2. 学费情况（Tuitions）
  3. 奖学金设置（Scholarships）
  4. 校园风采（Campus Life）
- **数据解耦**：当前阶段 UI 层只处理 Mock 数据，后续将对接基于 Cheerio 的爬虫数据和 Supabase 云端库。

## 4. 技术依赖与实现
- **组件库**：完全利用 Tailwind CSS 自定义 Drawer 的滑出动画和瀑布流网格，避免引入笨重的第三方 UI 库。
- **状态管理**：推荐在 `MapPage` 顶层使用 `useState` 控制 `isDrawerOpen` 和 `selectedContinent`，以确保平滑的数据传递。
